## 一、项目概述

### 1.1 项目背景

当前义齿工厂招聘系统已具备基础的应聘信息管理和面试预约功能，但缺乏完整的招聘流程管理模块。为实现从初试到录用的全流程数字化管理，需要开发招聘流程管理模块。

### 1.2 项目目标

构建一个完整的招聘流程管理模块，实现：
- ✅ 招聘流程全环节管理（初试、复试、录用、报到）
- ✅ 各环节数据流转与状态追踪
- ✅ 数据看板可视化呈现
- ✅ 数据筛选、导出及打印功能

### 1.3 方案选择说明

经过详细评估，采用**整合方案（单表结构）**，理由：
1. 流程固定，不需要分表的扩展性优势
2. 数据量适中（年千级），单表性能完全满足
3. 查询频繁，整合方案查询效率更高
4. 开发成本低，数据一致性易维护

## 二、数据库设计

### 2.1 表结构

#### 招聘流程主表（recruitment_process）

```sql
CREATE TABLE recruitment_process (
    id SERIAL PRIMARY KEY,
    
    -- ============================================
    -- 1. 基础信息（同步自applications表）
    -- ============================================
    application_id INTEGER REFERENCES applications(id) ON DELETE SET NULL,
    name VARCHAR(100) NOT NULL,
    gender VARCHAR(10),
    phone VARCHAR(20) NOT NULL,
    age INT,
    position VARCHAR(100),
    position_id INT,
    position_name VARCHAR(100),
    job_type VARCHAR(50),
    education VARCHAR(50),
    experience VARCHAR(50),
    skills TEXT,
    work_experience TEXT,
    salary_expectation VARCHAR(100),
    notes TEXT,
    dynamic_fields JSONB,
    source_status VARCHAR(20) DEFAULT 'pending',
    
    -- ============================================
    -- 2. 流程状态
    -- ============================================
    current_stage VARCHAR(50) NOT NULL DEFAULT 'first_interview',
    current_status VARCHAR(50) NOT NULL DEFAULT 'pending',
    
    -- ============================================
    -- 3. 初试环节信息
    -- ============================================
    first_interview_time TIMESTAMP,
    first_interviewer VARCHAR(100),
    first_interview_result VARCHAR(20),
    first_reject_reason VARCHAR(100),
    first_reject_detail TEXT,
    
    -- ============================================
    -- 4. 复试环节信息
    -- ============================================
    second_interview_time TIMESTAMP,
    second_interviewer VARCHAR(100),
    second_interview_result VARCHAR(20),
    second_reject_reason VARCHAR(100),
    second_reject_detail TEXT,
    hire_department VARCHAR(100),
    hire_position VARCHAR(100),
    job_level VARCHAR(50),
    hire_salary VARCHAR(100),
    hire_date DATE,
    
    -- ============================================
    -- 5. 报到环节信息
    -- ============================================
    is_reported VARCHAR(20),
    report_date DATE,
    no_report_reason VARCHAR(100),
    no_report_detail TEXT,
    
    -- ============================================
    -- 6. 系统字段
    -- ============================================
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    created_by VARCHAR(100),
    updated_by VARCHAR(100),
    source_type VARCHAR(20) DEFAULT 'sync',
    is_manual_add BOOLEAN DEFAULT FALSE
);
```

## 八、核心功能实现要点

### 8.1 数据加载

```javascript
class RecruitmentDataManager {
    constructor(supabaseClient) {
        this.client = supabaseClient;
        this.allData = [];
        this.filteredData = [];
    }
    
    async loadData() {
        const { data, error } = await this.client
            .from('recruitment_process')
            .select('*')
            .order('created_at', { ascending: false });
        
        if (error) throw error;
        
        this.allData = data || [];
        this.filteredData = [...this.allData];
        return this.allData;
    }
}
```

### 8.2 状态更新

```javascript
async function updateFirstInterview(id, formData) {
    const updateData = {
        first_interview_time: formData.first_interview_time,
        first_interviewer: formData.first_interviewer,
        first_interview_result: formData.first_interview_result
    };
    
    if (formData.first_interview_result === 'pass') {
        updateData.current_stage = 'second_interview';
        updateData.current_status = 'pending';
    } else {
        updateData.current_status = 'reject';
        updateData.first_reject_reason = formData.first_reject_reason;
        updateData.first_reject_detail = formData.first_reject_detail;
    }
    
    const { error } = await supabase
        .from('recruitment_process')
        .update(updateData)
        .eq('id', id);
    
    return { error };
}
```
