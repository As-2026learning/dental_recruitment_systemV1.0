## 一、项目背景与目标

### 1.1 项目背景

当前义齿工厂招聘系统已具备基础的应聘信息管理和面试预约功能，但缺乏完整的招聘流程管理模块。为实现从初试到录用的全流程数字化管理，需要开发招聘流程管理模块。

### 1.2 项目目标

构建一个完整的招聘流程管理模块，实现：
- ✅ 招聘流程全环节管理（初试、复试、录用、报到）
- ✅ 各环节数据流转与状态追踪
- ✅ 数据看板可视化呈现
- ✅ 数据筛选、导出及打印功能

## 二、功能需求规格

### 2.1 招聘流程管理

#### 2.1.1 流程环节定义

| 环节 | 状态码 | 说明 |
|------|--------|------|
| 初试 | `first_interview` | 第一轮面试筛选 |
| 复试 | `second_interview` | 第二轮面试评估 |
| 录用 | `hired` | 通过复试，进入录用阶段 |
| 报到 | `onboarded` | 已办理入职报到 |

#### 2.1.2 流程状态流转

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  初试   │───>│  复试   │───>│  录用   │───>│  报到   │
└────┬────┘    └────┬────┘    └────┬────┘    └────┬────┘
     │              │              │              │
     ▼              ▼              ▼              ▼
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 未通过  │    │ 未通过  │    │ 未报到  │    │ 已报到  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
```

### 2.2 初试模块功能

#### 2.2.1 数据同步
- 同步外部数据源或前置流程数据
- 自动关联应聘信息（姓名、电话、岗位、工种等）

#### 2.2.2 核心字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `first_interview_time` | DATETIME | 是 | 初试时间 |
| `first_interviewer` | VARCHAR(100) | 是 | 初试官 |
| `first_interview_result` | ENUM | 是 | 初试结果：通过/不通过 |
| `first_reject_reason` | TEXT | 条件必填 | 未通过原因 |

#### 2.2.3 初试结果选项

**通过状态**
- ✅ 通过 → 进入复试环节

**不通过状态及原因**
- ❌ 技能不符合要求
- ❌ 经验不足
- ❌ 薪资期望过高
- ❌ 稳定性差
- ❌ 其他（需手动补充详细说明）

### 2.3 复试模块功能

#### 2.3.1 数据获取规则
- 仅能获取并处理初试通过的候选人数据
- 自动继承初试阶段所有应聘信息

#### 2.3.2 继承字段

| 字段名 | 来源 | 说明 |
|--------|------|------|
| `position` | 应聘信息 | 应聘岗位 |
| `job_type` | 应聘信息 | 工种 |
| `name` | 应聘信息 | 姓名 |
| `phone` | 应聘信息 | 电话 |
| `education` | 应聘信息 | 学历 |
| `experience` | 应聘信息 | 工作经验 |

#### 2.3.3 新增字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `second_interviewer` | VARCHAR(100) | 是 | 复试官 |
| `second_interview_result` | ENUM | 是 | 复试结果：通过/不通过 |
| `second_reject_reason` | TEXT | 条件必填 | 未通过原因 |
| `hire_department` | VARCHAR(100) | 通过时必填 | 录用部门 |
| `hire_position` | VARCHAR(100) | 通过时必填 | 录用岗位 |
| `job_level` | VARCHAR(50) | 通过时必填 | 职级 |

#### 2.3.4 复试结果选项

**通过状态**
- ✅ 通过 → 进入录用环节

**不通过状态及原因**
- ❌ 专业能力不足
- ❌ 综合素质不符
- ❌ 团队协作能力差
- ❌ 薪资无法达成一致
- ❌ 其他（需手动补充详细说明）

### 2.4 录用模块功能

#### 2.4.1 数据获取规则
- 仅能获取并处理复试通过的候选人数据
- 自动继承复试阶段所有信息

#### 2.4.2 核心字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `is_reported` | ENUM | 是 | 是否报到：是/否 |
| `report_date` | DATE | 选择"是"时必填 | 报到日期 |
| `no_report_reason` | ENUM | 选择"否"时必填 | 未报到原因 |
| `no_report_detail` | TEXT | 选择"其他"时必填 | 未报到详细说明 |

#### 2.4.3 未报到原因选项

| 选项值 | 显示文本 | 是否需要补充说明 |
|--------|----------|------------------|
| `medical_fail` | 体检未通过 | 否 |
| `no_medical` | 未去体检 | 否 |
| `salary_issue` | 因薪资原因放弃 | 否 |
| `other` | 其他 | 是 |

### 2.5 数据看板要求

#### 2.5.1 综合数据看板

**统计指标卡片**
| 指标名称 | 说明 |
|----------|------|
| 总应聘人数 | 所有应聘记录数 |
| 初试人数 | 进入初试环节人数 |
| 初试通过率 | 初试通过人数 / 初试人数 |
| 复试人数 | 进入复试环节人数 |
| 复试通过率 | 复试通过人数 / 复试人数 |
| 录用人数 | 进入录用环节人数 |
| 实际报到人数 | 已报到人数 |
| 整体转化率 | 实际报到人数 / 总应聘人数 |

**流程漏斗图**
- 可视化展示各环节人数递减情况
- 显示各环节转化率

**趋势分析图表**
- 按日期统计各环节人数变化
- 支持日/周/月维度切换

#### 2.5.2 数据筛选功能

| 筛选条件 | 类型 | 说明 |
|----------|------|------|
| 应聘岗位 | 下拉选择 | 全部岗位/具体岗位 |
| 工种 | 下拉选择 | 全部工种/具体工种 |
| 流程环节 | 下拉选择 | 全部/初试/复试/录用/报到 |
| 时间范围 | 日期选择 | 开始日期-结束日期 |
| 结果状态 | 下拉选择 | 全部/通过/不通过/未处理 |

#### 2.5.3 数据导出功能

- 支持导出Excel格式
- 支持导出PDF格式
- 支持打印功能
- 可选择导出字段

## 三、数据模型设计

### 3.1 数据库表结构

#### 3.1.1 招聘流程表 (recruitment_process)

```sql
CREATE TABLE recruitment_process (
    id SERIAL PRIMARY KEY,
    application_id INTEGER REFERENCES applications(id),
    
    -- 当前流程环节
    current_stage VARCHAR(50) NOT NULL, -- first_interview/second_interview/hired/onboarded
    current_status VARCHAR(50) NOT NULL, -- pending/pass/reject
    
    -- 初试信息
    first_interview_time TIMESTAMP,
    first_interviewer VARCHAR(100),
    first_interview_result VARCHAR(20), -- pass/reject
    first_reject_reason VARCHAR(100),
    first_reject_detail TEXT,
    
    -- 复试信息
    second_interview_time TIMESTAMP,
    second_interviewer VARCHAR(100),
    second_interview_result VARCHAR(20), -- pass/reject
    second_reject_reason VARCHAR(100),
    second_reject_detail TEXT,
    hire_department VARCHAR(100),
    hire_position VARCHAR(100),
    job_level VARCHAR(50),
    
    -- 录用/报到信息
    is_reported VARCHAR(20), -- yes/no
    report_date DATE,
    no_report_reason VARCHAR(100),
    no_report_detail TEXT,
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 创建人/更新人
    created_by VARCHAR(100),
    updated_by VARCHAR(100)
);
```

#### 3.1.2 流程历史记录表 (recruitment_history)

```sql
CREATE TABLE recruitment_history (
    id SERIAL PRIMARY KEY,
    process_id INTEGER REFERENCES recruitment_process(id),
    application_id INTEGER REFERENCES applications(id),
    
    stage VARCHAR(50) NOT NULL, -- 环节
    action VARCHAR(50) NOT NULL, -- 操作类型
    old_status VARCHAR(50),
    new_status VARCHAR(50),
    operator VARCHAR(100),
    notes TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.2 实体关系图

```
┌─────────────────┐       ┌─────────────────────┐       ┌─────────────────┐
│   applications  │       │ recruitment_process │       │recruitment_history│
├─────────────────┤       ├─────────────────────┤       ├─────────────────┤
│ PK id           │──────>│ PK id               │<──────│ PK id           │
│    name         │       │ FK application_id   │       │ FK process_id   │
│    phone        │       │    current_stage    │       │ FK application_id│
│    position     │       │    current_status   │       │    stage        │
│    job_type     │       │    first_*          │       │    action       │
│    ...          │       │    second_*         │       │    operator     │
└─────────────────┘       │    hire_*           │       │    created_at   │
                          │    report_*         │       └─────────────────┘
                          │    created_at       │
                          │    updated_at       │
                          └─────────────────────┘
```
