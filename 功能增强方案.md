# 义齿工厂招聘系统功能增强方案

## 一、数据同步问题修复

### 问题描述
应聘登记及面试预约数据仅同步至后台"应聘信息管理"模块，未同步至"面试预约管理"模块。

### 解决方案
1. **统一数据模型**：确保 applications 表和 bookings 表之间的关联正确
2. **双向同步机制**：在创建/更新应聘信息时，自动同步到预约管理模块
3. **数据关联优化**：通过 application_id 或 phone 字段建立可靠关联

### 实现步骤
1. 检查 candidate-form-complete.html 中的数据提交逻辑
2. 确保提交时同时创建 applications 和 bookings 记录
3. 在 applications.html 中优化数据加载和关联逻辑

---

## 二、功能增强需求

### 2.1 应聘信息管理模块增强

#### a) 数据删除功能
- **需求**：允许管理员删除应聘记录
- **实现**：
  - 在每行数据后添加"删除"按钮
  - 点击后弹出确认对话框
  - 确认后从数据库删除记录
  - 同步删除关联的预约记录

#### b) 排序功能
- **需求**：支持按不同字段排序
- **实现**：
  - 点击表头进行排序
  - 支持排序字段：申请时间、姓名、岗位、面试日期
  - 支持升序/降序切换

### 2.2 面试预约管理模块增强

#### a) 数据删除功能
- **需求**：允许管理员删除预约记录
- **实现**：
  - 在每行数据后添加"删除"按钮
  - 点击后弹出确认对话框
  - 确认后从数据库删除记录

#### b) 排序功能
- **需求**：支持按不同字段排序
- **实现**：
  - 点击表头进行排序
  - 支持排序字段：预约日期、姓名、岗位、预约状态

### 2.3 字段管理排序功能

#### 需求
在后台"系统设置-模块管理"中，为字段增加排序功能，实现字段位置的手动调整。

#### 实现方案
1. **拖拽排序**：使用 HTML5 Drag and Drop API
2. **上下移动按钮**：为每个字段添加上移/下移按钮
3. **保存排序**：将排序结果保存到数据库的 sort_order 字段

### 2.4 招聘渠道补充信息功能

#### 需求
当用户选择"内部推荐"或"其他"选项时，需增加手动补充信息的输入框。

#### 实现方案
1. **动态显示**：根据选择动态显示/隐藏补充输入框
2. **字段命名**：
   - 内部推荐：显示"推荐人姓名"输入框
   - 其他：显示"请说明来源"输入框
3. **数据存储**：将补充信息保存到 form_data 中

## 三、技术实现方案

### 3.1 数据库表结构确认
需要确认以下表的存在和字段完整性：
- applications（应聘信息表）
- bookings（预约信息表）
- field_configs（字段配置表）
- field_options（字段选项表）
- template_field_mappings（模板字段映射表）

### 3.2 前端实现
- 使用原生 JavaScript 实现交互功能
- 使用 Supabase JavaScript 客户端进行数据库操作
- 保持与现有系统一致的 UI 风格

### 3.3 安全考虑
- 所有删除操作需要确认对话框
- 防止误删除重要数据
- 记录操作日志（可选）

## 四、测试用例

### 4.1 数据同步测试
1. 提交应聘登记 → 验证 applications 表有数据
2. 提交面试预约 → 验证 bookings 表有数据
3. 检查两个模块都能显示正确数据

### 4.2 删除功能测试
1. 删除应聘记录 → 验证数据被删除
2. 删除预约记录 → 验证数据被删除
3. 验证确认对话框正常工作

### 4.3 排序功能测试
1. 点击表头排序 → 验证数据按正确顺序显示
2. 再次点击切换排序方向 → 验证升序/降序切换

### 4.4 字段排序测试
1. 拖拽字段改变顺序 → 验证顺序改变
2. 刷新页面 → 验证顺序保持

### 4.5 招聘渠道测试
1. 选择"内部推荐" → 验证显示推荐人输入框
2. 选择"其他" → 验证显示说明输入框
3. 提交数据 → 验证补充信息正确保存
