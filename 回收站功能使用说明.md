# 回收站功能使用说明

## 功能概述

回收站功能实现了**软删除**机制，删除的数据不会立即从数据库中移除，而是被标记为"已删除"状态，可以在回收站中查看、恢复或彻底删除。

## 实施步骤

### 1. 数据库迁移（必须执行）

在 Supabase 控制台执行以下 SQL 文件：

```sql
-- 文件路径: sql/add-soft-delete-fields.sql
-- 为 applications、bookings、recruitment_process 表添加软删除字段

-- 1. 为 applications 表添加软删除字段
ALTER TABLE applications 
ADD COLUMN IF NOT EXISTS is_deleted BOOLEAN DEFAULT FALSE,
ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMP WITH TIME ZONE,
ADD COLUMN IF NOT EXISTS deleted_by TEXT;

-- 2. 为 bookings 表添加软删除字段
ALTER TABLE bookings 
ADD COLUMN IF NOT EXISTS is_deleted BOOLEAN DEFAULT FALSE,
ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMP WITH TIME ZONE,
ADD COLUMN IF NOT EXISTS deleted_by TEXT;

-- 3. 为 recruitment_process 表添加软删除字段
ALTER TABLE recruitment_process 
ADD COLUMN IF NOT EXISTS is_deleted BOOLEAN DEFAULT FALSE,
ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMP WITH TIME ZONE,
ADD COLUMN IF NOT EXISTS deleted_by TEXT;
```

## 功能特性

### 1. 软删除
- 删除数据时，数据被标记为 `is_deleted = true`
- 记录删除时间 `deleted_at` 和删除人 `deleted_by`
- 数据仍然保留在数据库中，可以恢复

### 2. 回收站查看
- **应聘信息页面**: 新增"回收站"标签页
- **招聘流程页面**: 点击"回收站"按钮打开模态框
- 显示所有已删除的记录，包括删除时间和删除人

### 3. 数据恢复
- 支持单条恢复和批量恢复
- 恢复后数据回到正常列表
- 保留原始数据完整性

### 4. 彻底删除
- 从数据库中永久删除数据
- 支持单条彻底删除和批量彻底删除
- 二次确认防止误操作

### 5. 清空回收站
- 一键清空所有已删除数据
- 需要二次确认

## 使用指南

### 在应聘信息页面

1. **删除数据**: 点击记录行的"删除"按钮 → 数据移动到回收站
2. **查看回收站**: 点击顶部的"🗑️ 回收站"标签页
3. **恢复数据**: 在回收站中点击"♻️ 恢复"按钮
4. **彻底删除**: 在回收站中点击"⚠️ 彻底删除"按钮
5. **批量操作**: 勾选多条记录 → 使用批量操作按钮

### 在招聘流程页面

1. **删除数据**: 选中记录 → 点击"批量删除"按钮
2. **查看回收站**: 点击工具栏右侧的"🗑️ 回收站"按钮
3. **恢复/删除**: 在弹出的模态框中操作

## 技术实现细节

### 数据库字段
- `is_deleted`: BOOLEAN - 标记是否已删除
- `deleted_at`: TIMESTAMP - 删除时间
- `deleted_by`: TEXT - 删除操作人

### API 方法
```javascript
// DataManager.js 新增方法
- deleteRecord(id, deletedBy)          // 软删除
- batchDeleteRecords(ids, deletedBy)   // 批量软删除
- restoreRecord(id)                    // 恢复单条
- batchRestoreRecords(ids)             // 批量恢复
- permanentDeleteRecord(id)            // 彻底删除
- batchPermanentDeleteRecords(ids)     // 批量彻底删除
- loadDeletedData()                    // 加载回收站数据
- emptyRecycleBin()                    // 清空回收站
```

## 注意事项

1. **必须先执行数据库迁移**，否则功能无法正常使用
2. 现有数据默认 `is_deleted = false`，不受影响
3. 彻底删除的数据**无法恢复**，请谨慎操作
4. 建议定期清理回收站，避免数据堆积
