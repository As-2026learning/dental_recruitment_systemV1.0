# 后台字段设置系统使用指南

## 系统概述

后台字段设置系统是一个功能完整的配置管理系统，用于对招聘申请表单的字段进行全面的编辑和自定义配置。该系统支持字段基础配置、动态字段配置、版本管理和岗位模板管理等核心功能。

## 系统功能

### 1. 核心功能模块
- **字段基础配置**：自定义字段的显示名称、数据类型、验证规则、必填项状态及显示顺序
- **动态字段配置**：基于应聘岗位的差异化字段配置功能
- **版本管理**：所有配置变更的持久化存储和完整的版本修改历史

### 2. 岗位模板管理
- **模板管理**：创建、编辑、删除多个独立的岗位模板
- **字段映射**：为每个岗位模板配置专属的字段集合，实现字段与岗位的多对多关联映射
- **批量操作**：提供岗位与字段配置的关联管理界面，支持批量操作

### 3. 前端交互功能
- **拖拽排序**：直观的拖拽排序界面，用于调整字段显示顺序
- **字段类型选择**：丰富的字段类型选择器，包括文本框、多行文本框、下拉框、单选按钮组、复选框组、日期选择器、数字输入框等
- **实时预览**：所见即所得地展示配置效果
- **岗位选择联动**：当用户在前端选择不同应聘岗位时，自动加载并渲染对应配置的字段集合

### 4. 数据验证系统
- **自定义验证规则**：针对特殊字段（如年龄、联系电话等）实现自定义验证规则配置
- **多种验证方式**：支持正则表达式验证、数值范围限制、长度限制等多种验证方式
- **实时表单验证**：当用户输入不符合规则时，即时显示明确的错误提示信息

## 使用指南

### 1. 字段配置

1. **创建新字段**：
   - 填写字段名称（英文标识符）
   - 填写显示标签（中文）
   - 选择字段类型
   - 设置是否必填
   - 配置验证规则（JSON格式）
   - 点击"创建字段"按钮

2. **编辑字段**：
   - 点击字段列表中的"编辑"按钮
   - 修改字段属性
   - 点击"保存更改"按钮

3. **管理字段选项**：
   - 点击字段列表中的"选项"按钮
   - 添加、编辑、删除选项
   - 拖拽调整选项顺序
   - 点击"保存选项"按钮

### 2. 岗位模板管理

1. **创建新模板**：
   - 填写模板名称
   - 填写模板描述
   - 设置是否激活
   - 点击"创建模板"按钮

2. **配置模板字段**：
   - 点击模板列表中的"配置字段"按钮
   - 从可用字段中添加字段到模板
   - 拖拽调整字段显示顺序
   - 点击"保存字段配置"按钮

3. **编辑模板**：
   - 点击模板列表中的"编辑"按钮
   - 修改模板属性
   - 点击"保存更改"按钮

### 3. 版本历史

- 查看所有配置变更的历史记录
- 了解每个变更的操作类型、时间和操作用户

### 4. 实时预览

1. 选择要预览的岗位模板
2. 系统自动生成并显示对应的表单
3. 测试字段验证规则和交互效果

## 技术架构

### 前端技术栈
- HTML5 + CSS3 + JavaScript
- Supabase SDK
- Sortable.js（拖拽排序）

### 后端技术栈
- Supabase（PostgreSQL）
- 行级安全策略

### 数据库表结构
- `field_configs` - 字段基础配置
- `position_templates` - 岗位模板
- `template_field_mappings` - 模板与字段关联
- `field_option_configs` - 字段选项配置
- `attachment_configs` - 选项附加信息配置
- `config_versions` - 配置版本历史

## 系统安全

1. **权限控制**：目前系统使用临时的匿名访问策略，生产环境应设置基于角色的访问控制
2. **数据安全**：所有配置操作都记录详细的版本历史，确保可追溯性
3. **输入验证**：前端和后端都实现了严格的输入验证，防止恶意输入

## 故障排除

### 常见问题

1. **数据库连接失败**：
   - 检查 Supabase URL 和 API Key 是否正确
   - 确保网络连接正常
   - 验证数据库表结构是否已正确创建

2. **字段选项不显示**：
   - 检查字段类型是否为下拉框、单选按钮组或复选框组
   - 确保已为字段添加选项

3. **模板字段排序不生效**：
   - 确保拖拽排序后点击"保存字段配置"按钮
   - 刷新页面查看最新排序

### 错误处理

- 系统会在操作失败时显示详细的错误信息
- 所有操作都会记录到版本历史中，便于排查问题
- 对于严重错误，系统会在控制台输出详细的错误日志

## 系统维护

1. **定期备份**：定期备份数据库配置，防止数据丢失
2. **版本管理**：利用系统的版本历史功能，定期清理过期的版本记录
3. **性能优化**：对于大量字段和模板的场景，考虑实现配置缓存机制
