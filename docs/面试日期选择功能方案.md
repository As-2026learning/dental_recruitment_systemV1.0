# 应聘信息综合管理页面 - 面试日期手动选择功能方案

## 一、现状分析

### 1.1 当前系统架构
- **前端页面**: `integrated-applications.html` - 应聘信息综合管理主页面
- **数据源**: Supabase数据库
  - `applications` 表 - 存储应聘信息
  - `bookings` 表 - 存储预约信息（包含面试日期）
- **数据关联**: 通过 `application_id` 或电话号码关联应聘信息和预约信息

### 1.2 现有功能
- 页面已包含面试日期筛选器（`filterBookingDate`）
- 表格显示面试日期列（从 `bookings.booking_date` 获取）
- 支持按面试日期排序

### 1.3 待实现功能
- 在表格行内直接编辑面试日期
- 日期选择器与系统UI风格统一
- 日期修改后即时保存到数据库

### 2.1 技术选型

| 组件 | 技术方案 | 理由 |

| 日期选择器 | 原生HTML5 `<input type="date">` | 无需引入第三方库，与现有代码保持一致 |

#### 步骤3: JavaScript功能实现

**A. 新增编辑状态管理变量**
```javascript
// 当前正在编辑的行ID
var editingBookingId = null;
var editingApplicationId = null;
```

**B. 新增核心函数**

1. **进入编辑模式**
```javascript
function editInterviewDate(applicationId, bookingId, currentDate) {
    editingApplicationId = applicationId;
    editingBookingId = bookingId;
    renderTable(); // 重新渲染表格以显示编辑状态
}
```

2. **保存面试日期**
```javascript
async function saveInterviewDate(newDate) {
    try {
        // 如果已有预约记录，更新日期
        if (editingBookingId) {
            await supabaseClient
                .from('bookings')
                .update({ booking_date: newDate })
                .eq('id', editingBookingId);
        } else {
            // 如果没有预约记录，创建新预约
            await supabaseClient
                .from('bookings')
                .insert({
                    application_id: editingApplicationId,
                    booking_date: newDate,
                    status: 'pending'
                });
        }
        
        // 刷新数据
        await loadData();
        editingBookingId = null;
        editingApplicationId = null;
        
        alert('面试日期保存成功！');
    } catch (e) {
        alert('保存失败: ' + e.message);
    }
}
```

3. **取消编辑**
```javascript
function cancelEdit() {
    editingBookingId = null;
    editingApplicationId = null;
    renderTable();
}
```

**C. 表格渲染逻辑改造**
在 `renderTable()` 函数中修改面试日期单元格渲染逻辑：
```javascript
// 判断当前行是否处于编辑状态
const isEditing = editingApplicationId === app.id;

if (isEditing) {
    html += `
        <td>
            <div class="inline-date-picker">
                <input type="date" id="editDate_${app.id}" value="${app.booking?.booking_date || ''}">
                <button class="btn-save" onclick="saveInterviewDate(document.getElementById('editDate_${app.id}').value)">✓</button>
                <button class="btn-cancel" onclick="cancelEdit()">✗</button>
            </div>
        </td>
    `;
} else {
    html += `
        <td>
            <span>${bookingDate}</span>
            <button class="btn-edit-date" onclick="editInterviewDate('${app.id}', '${app.booking?.id}', '${bookingDate}')">编辑</button>
        </td>
    `;
}
```

```

```

**新数据流：**
```

                                ↑
                                │ 新增：更新booking_date
                         用户操作
```

### 3.4 数据结构变更

无需修改数据库表结构，复用现有 `bookings` 表：
- `booking_date`: date类型，存储面试日期
