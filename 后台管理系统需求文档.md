## 一、项目背景与目标

### 1.1 项目背景

义齿工厂需要一个后台管理系统来管理应聘信息采集表单。当前系统存在以下问题：
- 模块导航不稳定，经常出现无法跳转的情况
- 代码结构复杂，维护困难
- 功能分散，用户体验不佳

### 1.2 项目目标

重新开发一个稳定、简洁、易用的后台管理系统，实现：
- ✅ 字段配置管理
- ✅ 岗位模板管理
- ✅ 岗位管理
- ✅ 实时预览功能
- ✅ 版本历史记录

## 二、功能需求

### 2.1 字段配置管理

#### 2.1.1 功能描述
管理应聘表单中使用的所有字段，包括字段的创建、编辑、删除。

#### 2.1.2 字段属性

| 属性名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 字段名称 | 文本 | 是 | 英文标识，如：name, phone |
| 显示标签 | 文本 | 是 | 中文显示名称，如：姓名、联系电话 |
| 字段类型 | 枚举 | 是 | 见下方字段类型列表 |
| 是否必填 | 布尔 | 否 | 默认为否 |
| 占位符 | 文本 | 否 | 输入框提示文字 |
| 默认值 | 文本 | 否 | 字段默认填充值 |
| 验证规则 | JSON | 否 | 数据验证规则配置 |

#### 2.1.3 支持的字段类型

| 类型 | 说明 | 示例 |
|------|------|------|
| text | 单行文本框 | 姓名、地址 |
| textarea | 多行文本框 | 自我介绍 |
| number | 数字输入框 | 年龄、工作年限 |
| select | 下拉选择框 | 学历、工作经验 |
| radio | 单选按钮组 | 性别 |
| checkbox | 复选框组 | 技能特长 |
| date | 日期选择器 | 出生日期 |

#### 2.1.4 功能操作

1. **创建字段**
   - 填写字段基本信息
   - 选择字段类型
   - 设置是否必填
   - 保存到数据库

2. **编辑字段**
   - 修改字段属性
   - 更新数据库记录
   - 记录版本历史

3. **删除字段**
   - 确认删除操作
   - 从数据库移除
   - 记录版本历史

4. **字段选项管理**（针对select/radio/checkbox类型）
   - 添加选项
   - 编辑选项
   - 删除选项
   - 拖拽排序
   - 设置选项附加信息（如内部介绍需要填写介绍人信息）

---

### 2.2 岗位模板管理

#### 2.2.1 功能描述
管理不同岗位的表单模板，每个模板包含一组字段配置。

#### 2.2.2 模板属性

| 属性名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 模板名称 | 文本 | 是 | 如：学徒模板、技工模板 |
| 模板描述 | 文本 | 否 | 模板用途说明 |
| 是否激活 | 布尔 | 否 | 默认为激活状态 |
| 创建时间 | 时间戳 | 自动 | 系统自动生成 |
| 更新时间 | 时间戳 | 自动 | 系统自动生成 |

#### 2.2.3 功能操作

1. **创建模板**
   - 填写模板名称和描述
   - 设置激活状态
   - 保存到数据库

2. **编辑模板**
   - 修改模板信息
   - 更新数据库记录

3. **删除模板**
   - 确认删除操作
   - 从数据库移除

4. **模板字段配置**
   - 从字段库中选择字段
   - 添加到模板
   - 设置字段在模板中的显示顺序（拖拽排序）
   - 设置字段是否启用
   - 从模板中移除字段

---

### 2.3 岗位管理

#### 2.3.1 功能描述
管理招聘的具体岗位，每个岗位关联一个模板。

#### 2.3.2 岗位属性

| 属性名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 岗位名称 | 文本 | 是 | 如：学徒、牙科技工 |
| 关联模板 | 外键 | 是 | 关联的岗位模板 |
| 排序顺序 | 数字 | 否 | 显示排序，数字越小越靠前 |
| 是否激活 | 布尔 | 否 | 是否在前端显示 |
| 创建时间 | 时间戳 | 自动 | 系统自动生成 |

#### 2.3.3 功能操作

1. **创建岗位**
   - 填写岗位名称
   - 选择关联模板
   - 设置排序顺序
   - 保存到数据库

2. **编辑岗位**
   - 修改岗位信息
   - 更换关联模板
   - 更新数据库记录

3. **删除岗位**
   - 确认删除操作
   - 从数据库移除

4. **查看岗位列表**
   - 显示所有岗位
   - 显示关联的模板名称
   - 支持按排序顺序排列

---

### 2.4 实时预览功能

#### 2.4.1 功能描述
预览不同岗位对应的表单效果，确保配置正确。

#### 2.4.2 功能操作

1. **选择岗位**
   - 从下拉列表选择要预览的岗位
   - 系统自动加载该岗位关联的模板字段

2. **表单预览**
   - 根据模板字段配置动态生成表单
   - 显示字段标签、类型、必填标识
   - 支持下拉选项、单选、多选等交互元素
   - 模拟真实表单外观

3. **预览限制**
   - 预览模式下表单不可提交
   - 点击提交按钮提示"预览模式"

---

### 2.5 版本历史记录（可选增强功能）

#### 2.5.1 功能描述
记录所有配置的变更历史，便于追溯和回滚。

#### 2.5.2 记录内容

| 字段 | 说明 |
|------|------|
| 实体类型 | 字段/模板/映射 |
| 实体ID | 被操作的数据ID |
| 操作类型 | 创建/更新/删除 |
| 操作前数据 | JSON格式 |
| 操作后数据 | JSON格式 |
| 操作时间 | 时间戳 |
| 操作用户 | 执行操作的用户 |

#### 2.5.3 功能操作

1. **查看历史**
   - 按时间倒序显示所有操作记录
   - 显示操作类型、时间、用户

2. **筛选查询**
   - 按实体类型筛选
   - 按操作类型筛选
   - 按时间范围筛选

## 五、数据模型

### 5.1 数据库表结构

#### 5.1.1 field_configs（字段配置表）

```sql
CREATE TABLE field_configs (
    id SERIAL PRIMARY KEY,
    field_name VARCHAR(100) NOT NULL,      -- 字段名称（英文）
    field_label VARCHAR(200) NOT NULL,     -- 显示标签（中文）
    field_type VARCHAR(50) NOT NULL,       -- 字段类型
    is_required BOOLEAN DEFAULT FALSE,     -- 是否必填
    placeholder VARCHAR(200),              -- 占位符文本
    default_value TEXT,                    -- 默认值
    validation_rules JSONB,                -- 验证规则（JSON格式）
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

#### 5.1.2 position_templates（岗位模板表）

```sql
CREATE TABLE position_templates (
    id SERIAL PRIMARY KEY,
    template_name VARCHAR(200) NOT NULL,   -- 模板名称
    description TEXT,                      -- 模板描述
    is_active BOOLEAN DEFAULT TRUE,        -- 是否激活
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

#### 5.1.3 positions（岗位表）

```sql
CREATE TABLE positions (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,            -- 岗位名称
    template_id INTEGER REFERENCES position_templates(id), -- 关联模板
    sort_order INTEGER DEFAULT 0,          -- 排序顺序
    is_active BOOLEAN DEFAULT TRUE,        -- 是否激活
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

#### 5.1.4 template_field_mappings（模板字段映射表）

```sql
CREATE TABLE template_field_mappings (
    id SERIAL PRIMARY KEY,
    template_id INTEGER REFERENCES position_templates(id),
    field_id INTEGER REFERENCES field_configs(id),
    sort_order INTEGER DEFAULT 0,          -- 字段在模板中的排序
    is_enabled BOOLEAN DEFAULT TRUE,       -- 是否启用
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(template_id, field_id)
);
```

#### 5.1.5 field_option_configs（字段选项配置表）

```sql
CREATE TABLE field_option_configs (
    id SERIAL PRIMARY KEY,
    field_id INTEGER REFERENCES field_configs(id),
    option_value VARCHAR(200) NOT NULL,    -- 选项值
    option_label VARCHAR(200) NOT NULL,    -- 选项显示文本
    sort_order INTEGER DEFAULT 0,          -- 排序
    has_attachments BOOLEAN DEFAULT FALSE, -- 是否有附加信息
    created_at TIMESTAMP DEFAULT NOW()
);
```

#### 5.1.6 config_versions（版本历史表）- 可选

```sql
CREATE TABLE config_versions (
    id SERIAL PRIMARY KEY,
    entity_type VARCHAR(50) NOT NULL,      -- 实体类型：field/template/mapping
    entity_id INTEGER NOT NULL,            -- 实体ID
    operation VARCHAR(50) NOT NULL,        -- 操作类型：create/update/delete
    old_data JSONB,                        -- 操作前数据
    new_data JSONB,                        -- 操作后数据
    user_id VARCHAR(100),                  -- 操作用户
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 七、技术架构建议

### 7.1 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 纯HTML + CSS + JavaScript | 无需框架，简单直接 |
| 后端 | Supabase (BaaS) | 后端即服务，简化开发 |
| 数据库 | PostgreSQL | Supabase自带 |
| 部署 | 静态文件托管 | 简单部署 |

### 7.2 架构特点

1. **单页面应用（SPA）**
   - 所有功能在一个页面内完成
   - 通过JavaScript切换显示不同模块
   - 无需页面跳转

2. **前后端分离**
   - 前端负责界面展示
   - Supabase负责数据存储和API
   - 通过RESTful API通信

3. **无服务器架构**
   - 使用Supabase托管数据库
   - 无需自己搭建服务器
   - 降低运维成本
